# Космический хакатон 2026

Приветствуем участников космического хакатона. В этом репозитории вы сможете найти задачи и материалы, необходимые для их решения. Желаем всем участникам удачи!

## Содержание репозитория

- В корне репозитория содержатся три директории: `Problem_1`, `Problem_2` и `Problem_3`. Внутри них лежат вся необходимая информация о задачах;
- В каждой из указанных выше директории содержится файл `problem_info.md`. В нём находятся условия задачи, описание выходных данных и список источников, которые могут помочь участникам решить задачу;
- Также внутри этих директорий вы можете найти директорию `problem_data`. Там будут содержаться все необходимые для решения картинки и файлы. Так же там может лежать литература, которая была указана в источниках.

## О задачах

В этом году участников ждут три задачи, отражающие реальные вызовы космической отрасли — от проектирования миссий до обработки спутниковых данных.

**Первая задача** — проектирование комбинированной схемы выведения на геостационарную орбиту. Нужно найти оптимальную промежуточную орбиту и закон управления электроракетной двигательной установкой, чтобы за минимальное время доставить на ГСО спутник заданной массы.

**Вторая задача** — анализ аварии космического лифта. Что произойдёт при обрыве троса? Накрутится ли он на планету или сгорит в атмосфере? Участники должны самостоятельно разбить проблему на этапы и провести численное моделирование ключевого сценария.

**Третья задача** — восстановление карты местности под облаками. По серии аэроснимков, где облака закрывают разные участки, нужно собрать единое изображение, научившись выделять как плотную, так и полупрозрачную облачность.

Задачи не имеют единственного верного решения и допускают разные подходы — ценятся инженерная смекалка, чистота кода и обоснованность выводов.

## Специальная задача от партнера

Приятный сюрприз от наших друзей — РКК “Энергия”! 🚀

В этом году они подготовили для вас бонусную задачу вне конкурсного зачета. Она не добавит баллов, зато подарит эмоции, а главное — лучших в её решении ждет специальный приз от легенд отечественной космонавтики. Уровень сложности — космический, награда — уникальная. 

**Специальная задача** — построение местной вертикали по данным инфракрасного датчика. Спутник определяет своё положение относительно Земли по пересечению сканирующих лучей с горизонтом. Нужно разработать алгоритм, который по засечкам углов восстановит вектор на центр Земли в системе координат прибора и вычислит углы тангажа и крена. Алгоритм следует проверить на реальных тестовых случаях, визуализировать результат и сделать выводы о его точности и устойчивости.

Познакомиться с задаче подробнее можно в [Problem_extra](Problem_extra/)

## Как оформлять решение

Для начала сделайте fork этого репозитория в ваш GitHub аккаунт. Вся работа ведётся **внутри вашего репозитория**, ссылку на него вам необходимо будет отправить для проверки решений. Пожалуйста, сохраняйте структуру репозитория: все работы над конкретной задачей должны вестись в папке для соответствующей задачи.

Для каждой задачи необходимо приложить pdf или md файл с описанием вашего решения. В нём должны быть:
- Мотивация. Ответьте на вопрос: почему вы выбрали именно такой метод для решения задачи?
- Математическая и физическая модель. Опишите, какими формулами и законами вы пользовались, какой формализм помог вам получить ваше решение. Не расписывайте эти моменты слишком подробно, оставьте лишь то, что по вашему мнению самое важное для понимания
- Описание технической реализации. Чем более понятно описан ваш алгоритм, тем проще экспертам будет понять его и оценить по достоинству.

За час до окончания времени на решения, в 18:00, откроется форма для загрузки решений. Туда необходим будет загрузить ссылку на ваш GitHub и основные метрики, специфичные для каждой задачи. В 19:00 форма закроется.

## Презентация для защиты решения в финальный день
По результатам основного этапа хакатона будет выбрано 5 лучших команд, которые будут защищаться в финале. Вам будет необходимо подготовить презентацию в формате pptx или pdf. С этой презентацией будет проходить защита ваших решений на финальном этапе в воскресенье (12.04).

На защиту вам будет выделено 15 минут + 5-7 минут на вопросы от жюри и других команд. Создавая презентацию, держите эту цифру в голове, иначе есть шанс, что вас прервут по истечению этого времени. В презентации должны быть отражены следующие моменты:
- Мотивация (см. выше);
- Математическая и физическая модель (см. выше);
- Описание технической реализации (см. выше);
- Идеи по улучшению решения. Порассуждайте, как можно улучшить ваши расчёты или мат модели, чего не хватило, чтобы получить более точную картину и т.п.

Пожалуйста, подготовьте черновик презентации **до отправки итогового решения**. Вы сможете дополнить её после отправки и отправить нам итоговый вариант за час до начала финального этапа.
